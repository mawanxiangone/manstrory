#             新型城域网BRAS池化部署方案及实践        

​                            2021-06-21                                        作者：中兴通讯   高素勤                                        阅读量：                366                    

随着5G建设的全面启动，运营商作为国家信息化建设的主力军，正迎来前所未有的新发展机遇。在“云改数转”的浪潮下，中国电信提出打造云网融合的新型基础设施的设想：以打造融合、敏捷、简洁、云化、智能、安全的网络为目标，采用“积木式”架构、模块化组件，坚持多业务融合承载、多厂商异构组网，构建架构弹性扩展、云网标准化对接、业务快速提供的新型城域网络。

在新型城域网架构下，需要保障用户固网和无线接入体验的一致性，满足未来个人用户和政企、垂直行业、物联网业务的海量连接及高体验需求。位于城域网边缘的BRAS（Broadband Remote Access  Server）设备作为实现各种用户业务的入口，是新型城域网建设的重要一环，面对网络向云化、智能化转型的趋势，传统BRAS及其分布式的部署方式显得力不从心，其控制与转发紧耦合的架构模式已无法满足层出不穷的新业务、新流量。因此，构建一个业务丰富、运维简化、资源集约、稳定可靠的城域网边缘已迫在眉睫。

## 中兴通讯新型城域网BRAS池化部署方案

中兴通讯vBRAS产品采用转控分离架构，vBRAS-C（vBRAS控制面）采用X86云化技术实现控制面虚拟化，关键模块包括MP（Main Plane）、 PP（Protocol Processor）、LB（Load Balancer）、ARU（Arbitration  Unit）等，负责BRAS业务的控制与管理，如用户接入控制、协议报文处理、用户AAA管理以及地址管理等功能。vBRAS-U（vBRAS转发面）采用基于NP（Network Processor）的高性能转发池，负责执行用户策略、流量转发和流量统计等。控制面和转发面之间IP可达即可。

基于ZXR10 V6000  vBRAS产品的中兴通讯新型城域网BRAS池化部署方案，克服了传统BRAS设备业务控制层资源利用率不均、新业务部署周期长、分散配置运维工作量大的缺点。方案采用SDN&NFV技术，控制转发分离、软硬件解耦，从而实现了控制面云化、新业务快速灵活部署、配置运维集中控制，降低了运营商OPEX和CAPEX，助力运营商构建超大容量、全面可靠、极简运维、平滑演进的新型城域网。

- 超大容量，规模部署：方案采用转发控制分离架构，克服了当前BRAS设备控制面资源匮乏的问题；单系统支持2000万并发会话，单控制面支持200个以上转发设备；高性能转发池提供单槽400G转发能力、1T演进；覆盖城域网所有业务类型。

- 全面可靠，业务永续：业界首家支持控制面跨DC热备；所有控制面组件冗余备份，单个组件故障时无缝倒换；转发面天然池化保护，业务负载均衡，主用转发设备发生故障时，自动切换至备用设备，用户无需重新拨号；全方位打造业务永续的系统。

- 集中配置，极简运维：控制面集中管控，控制面自动下发配置到转发面，降低75%的宽带业务配置工作量；新业务上线涉及到软件升级时只需对控制面进行升级，上线时间从数月缩减为1周。

- 平滑演进，保护投资：无需改动网络现有架构，可直接部署于当前的城域骨干网中，支持向未来以DC为核心的网络无缝演进；高性能转发设备可由现网BRAS设备升级演进，利旧现有网元资产，充分保护客户既有投资。

## 实践与应用

2020年，中兴通讯和中国电信在多地市进行了BRAS池化方案的部署实践和能力验证，助力中国电信城域网边缘的智能化、云化转型。目前，BRAS池化部署方案分为VPWS（Virtual Private Wire Service）接入方案和VPLS（Virtual Private LAN  Service）接入方案。这两种业务组网方案及对应的U面池化保护方案在试点中均有实践（见图1）。

### VPWS接入方案

OLT（Optical Line  Terminal）不捆绑上行双归一对A-Leaf（Access Leaf），加入不同的VPWS（Virtual Private Wire  Service）实例，即每个OLT对应2个VPWS实例；vBRAS-U用户侧接口捆绑后双归接入到不同的S-Leaf（Service  Leaf）。每台UP（User Plane）上子接口只接入其中1个VPWS实例；S-Leaf部署双归双活。此方案中，UP（User  Plane）可实现子接口粒度的切换，即基于每OLT粒度的切换。但由于VPWS是基于点到点方案实现OLT与UP的连接，所以OLT侧的故障会联动UP侧的主备倒换，需要端到端故障线性连通检测。此外，VPWS方案无法实现负荷分担，会造成OLT上行带宽需求翻倍，需人工规划，配置工作量大，逻辑关系复杂，维护难度较高。

### VPLS接入方案

OLT捆绑双归接入一对A-Leaf，根据site或者service划分VPLS实例；一对A-Leaf针对同一个OLT的捆绑链路配置相同的ESI；每台UP用户侧接口SG（Smartgroup）捆绑双归S-Leaf，每SG口进入同一个VPLS实例。S-Leaf部署双归双活。此方案中，BRAS池可实现真正池化，做到N+1温备。该方案配置和维护简单，OLT侧的故障不会联动UP侧的倒换。但由于Leaf节点都需要学习用户终端的MAC地址，S-Leaf的MAC地址表容量压力较大，同时需要考虑VPLS实例内的广播攻击和非法MAC攻击。

为了解决VPLS接入方案面临的难题，电信集团携手设备厂商推出UMR（Unknown MAC  Route）方案。其BRAS池化接入原理和VPLS方案完全一样，但A-Leaf仅给S-Leaf发送UMR，不发送其他MAC路由，如此大大减少了S-Leaf设备对MAC地址容量的需求和RR（Route  Reflector）通告MAC地址的压力。此方案可实现单台OLT与多台UP间的灵活归属，以及UP面灵活的N+1保护，是未来新型城域网BRAS池化部署的趋势。

中兴通讯新型城域网BRAS池化方案聚焦运营商网络和业务的特点，既满足电信级运营多会话、大带宽、高可靠和高效率的需要，又充分考虑到运营商网络的开放性及演进需求；通过控制面和转发面的分离，高性能转发面可以满足未来4K/8K、AR/VR大视频业务的高性能要求，控制面可以满足能力开放、维护简便、业务灵活等要求，完全适应未来城域网虚拟化的发展，为宽带业务的运营带来巨大价值。
